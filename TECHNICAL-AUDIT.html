02 — Security

Security Implementation

Every recommended HTTP security header is deployed and configured for maximum protection. This configuration achieves an A+ rating on industry-standard security audits.

✓
HSTS
Strict-Transport-Security
Forces browsers to use HTTPS for all future requests, preventing protocol downgrade attacks and cookie hijacking.
✓
Content Security Policy
Content-Security-Policy
Mitigates cross-site scripting (XSS) and data injection attacks by whitelisting approved content sources.
✓
Frame Protection
X-Frame-Options
Prevents the site from being embedded in iframes, blocking clickjacking attacks.
✓
MIME Sniffing Guard
X-Content-Type-Options
Stops browsers from guessing content types, preventing MIME-type confusion attacks.
✓
XSS Filter
X-XSS-Protection
Activates legacy browser XSS filters as an additional layer of scripting defense.
✓
Referrer Control
Referrer-Policy
Controls how much referrer information is shared when navigating away, protecting user privacy.
✓
Permissions Policy
Permissions-Policy
Restricts access to device APIs (camera, microphone, geolocation) preventing unauthorized feature usage.
✓
Cross-Origin Opener
Cross-Origin-Opener-Policy
Isolates the browsing context to prevent cross-origin attacks like Spectre side-channel exploits.
✓
Cross-Origin Resources
Cross-Origin-Resource-Policy
Prevents other origins from loading resources, blocking unauthorized cross-origin data leaks.
✓
DNS Prefetch Control
X-DNS-Prefetch-Control
Controls DNS prefetching to prevent information leaks about which links exist on the page.
✓
Bot & Spam Protection
Honeypot Field
Invisible form fields trap automated bots, blocking spam submissions without affecting real users.
✓
HTTPS Enforced
TLS / SSL Certificate
All traffic is encrypted in transit. HTTP requests are permanently redirected to HTTPS.
✓
GDPR Cookie Consent
Cookie Banner
Fully compliant cookie consent banner with accept/reject options, respecting user privacy preferences before loading any tracking scripts.
✓
Privacy Policy (Tietosuojaseloste)
GDPR Article 13/14
Comprehensive Finnish-language privacy policy documenting data collection, processing, user rights, and contact information for data requests.
✓
Service Worker (PWA)
sw.js + manifest.json
Progressive Web App with offline support, caching strategy, and mobile install capability for app-like experience on any device.

04 — Performance

Performance Optimizations

Every optimization has been applied to ensure fast load times, low bandwidth usage, and top Core Web Vitals scores.

⚡
Lazy Loading
All below-fold images use native lazy loading, reducing initial page weight and improving Largest Contentful Paint (LCP).
▶
Video Preload Optimization
Video elements configured with metadata preloading to prevent blocking the main thread during initial page render.
📦
Cache Headers (1 Year)
Static assets configured with long-term cache headers (max-age=31536000), minimizing repeat downloads for returning visitors.
🌐
DNS Prefetch
External resource domains are prefetched, reducing DNS lookup latency for third-party resources by ~50ms per request.
AA
Font Display Swap
All web fonts use display:swap to prevent invisible text during loading, improving First Contentful Paint (FCP).
✨
Zero Heavy Frameworks
Built with vanilla HTML/CSS/JS — no React, Angular, or jQuery. Minimal dependencies ensure fastest possible load times.
📷
PWA with Service Worker
Full offline support via service worker caching strategy. Users can continue browsing even without an internet connection.
🔗
Resource Hints
Preconnect and dns-prefetch hints for critical third-party origins, reducing connection setup time by up to 100ms.
📸
Hero Image Priority
fetchpriority="high" on hero images ensures the largest contentful paint (LCP) element loads as early as possible.
📱
Mobile Install (manifest.json)
Web app manifest enables "Add to Home Screen" on mobile devices, providing an app-like experience with custom icons and splash screen.

06 — Accessibility

Accessibility (WCAG 2.1 AA)

The website meets WCAG 2.1 Level AA standards, ensuring it is usable by people with disabilities and accessible across all assistive technologies.

✓
Skip Navigation Link
A skip-to-content link allows keyboard users to bypass repeated navigation and jump directly to the main content area.
✓
ARIA Roles on All Landmarks
Semantic ARIA landmark roles (banner, navigation, main, contentinfo) applied to all major page regions for screen reader navigation.
✓
aria-labels on Interactive Elements
All buttons, links, and form controls have descriptive aria-labels ensuring screen readers convey their purpose accurately.
✓
aria-expanded on Mobile Menu
The mobile hamburger menu communicates its open/closed state via aria-expanded, providing clear feedback to assistive technology users.
✓
Screen Reader Only Class
A visually-hidden utility class provides additional context to screen reader users without affecting the visual design.
✓
Keyboard Navigation
All interactive elements are fully operable via keyboard. Focus indicators are visible and logical tab order is maintained throughout.
✓
Print Stylesheet
A dedicated print stylesheet ensures the content is readable and well-formatted when printed, hiding navigation and non-essential elements.
